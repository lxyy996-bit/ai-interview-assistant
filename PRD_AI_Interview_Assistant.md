# AI 智能面试助手 - 产品需求文档 (PRD)

> 本文档用于指导 Kimi Code 进行代码开发

---

## 1. 产品概述

### 1.1 产品定位
AI 驱动的简历分析与模拟面试系统，通过整合公司情报与 STAR 方法论，为用户提供精准的面试准备方案。

### 1.2 核心价值
- **公司情报驱动**：基于目标公司近期公开信息（新闻、财报）定制分析
- **STAR 结构化输出**：每道题都提供 S-T-A-R 回答框架 + 示例
- **动态适配**：不同公司、不同岗位生成不同问题

### 1.3 输入输出
| 类型 | 内容 |
|------|------|
| **输入** | 简历 + 目标岗位 + 目标公司 + 所在地 |
| **输出** | 智能评分 + 定制化模拟面试题（STAR 法则） |

---

## 2. MVP 功能模块

| 模块 | 功能说明 | 优先级 |
|------|----------|--------|
| 简历解析 | 支持 PDF/Word 格式，提取关键信息（技能、经历、项目） | P0 |
| 公司情报 | 自动抓取官网、新闻、财报等公开信息 | P0 |
| 智能评分 | 匹配度 + 完整性 + 关键词优化评分 | P0 |
| 模拟面试 | 生成 5 道个性化 STAR 面试问题 | P0 |
| 优化建议 | 针对目标公司的简历改进方案 | P1 |

## 4. 接口定义

### 4.1 第一阶段：搜索关键词生成

**接口名称**: `generate_search_keywords`

**输入参数**:
```typescript
interface SearchKeywordsInput {
  company: string;      // 目标公司名称
  job_name: string;     // 岗位名称
  jd: string;           // 岗位要求/描述
  city: string;         // 工作地点
}
```

**System Prompt**:
```
你是一名精通信息检索的资深猎头。你的任务是根据用户提供的公司和岗位，生成3组极其精准的搜索关键词，帮助用户挖掘该岗位的"隐藏招聘需求"和公司的"业务风向"。

输出要求：
- 禁任何废话，只输出关键词
- 每组关键词用空格分隔
- 必须包含：1. 战略财务类；2. 业务变动类；3. 员工口碑/避雷类
- 请生成包含具体财报年份的搜索词，例如：美团 2024 财报 优选业务 亏损收窄
```

**User Prompt 模板**:
```
目标公司：{{company}}
岗位名称：{{job_name}}
岗位要求：{{jd}}
工作地点：{{city}}

请基于以上信息生成搜索词：
战略与财务：[生成关于 {{company}} 2025 战略布局、营收重点、核心项目的搜索词]
业务与变动：[生成关于 {{company}} 最近一年架构调整、高管变动、业务收缩或扩张的搜索词]
口碑与避雷：[生成关于 {{company}} {{jd}} {{job_name}} 加班、面试坑点、职场氛围的搜索词]
```

**输出格式**:
```
战略与财务：关键词1 关键词2 关键词3 ...
业务与变动：关键词1 关键词2 关键词3 ...
口碑与避雷：关键词1 关键词2 关键词3 ...
```

### 4.3 第二阶段：深度分析与 STAR 生成

**接口名称**: `generate_interview_analysis`

**输入参数**:
```typescript
interface InterviewAnalysisInput {
  company: string;          // 目标公司
  job_name: string;         // 岗位名称
  jd: string;               // 岗位要求
  resume: string;           // 解析后的简历文本
  search_info: {
    baidu_organic: string;  // 百度搜索结果摘要
    bing_organic: string;   // 必应搜索结果摘要
  };
}
```

**System Prompt**:
```
# 角色
你是一位全球顶尖的首席人力资源官（CHRO）和商业战略分析师。你擅长通过零散的公司动态分析其背后的业务逻辑，并据此评估人才的"战略契合度"。

# 任务核心
1. 解读战略：从搜索情报中判断公司目前处于"业务扩张期"、"降本增效期"还是"业务转型期"
2. 洞察需求：分析该公司招聘该岗位的底层逻辑（是填补空缺、还是为新业务储备领军人才）
3. 深度评估：对比用户简历与"战略需求"的匹配度，而非简单的关键词匹配
4. STAR 访谈引导：针对简历中最弱但公司最看重的环节，设计深度追问

# 处理规则
- 如果搜索结果包含大量无用信息，优先提取关于公司"降本增效"、"业务收缩/扩张"、"财务盈亏"的关键词
- 若确实无直接动态，必须结合该行业 2025-2026 年的公认趋势进行推演，严禁仅给出通用回答
- 如果搜索结果为空或信息不足，请基于该行业的通用现状进行分析，并诚实告知用户"未获取到近期特定动态"
```

**User Prompt 模板**:
```
1. 目标背景
公司：{{company}}
岗位名称：{{job_name}}
岗位要求：{{jd}}

2. 搜集到的实时情报
{{baidu.organic}}、{{bing.organic}}

3. 用户当前简历
{{resume}}

请按以下格式输出分析报告：

## 公司战略意图解码
**业务现状**：[基于搜索结果总结公司近况]

**岗位"潜台词"**：[公司为什么现在招这个人？最核心要解决什么问题？]

## ATS 战略匹配评分
**当前分值**：[0-100 分]

**优势点**：[简历中哪些经历最能打动该司？]

**致命缺口**：[缺少哪些决定性的能力或关键词？]

## 简历修改意见
**修改逻辑**：[例如：将通用的"项目管理"改为"在高压降本环境下的资源整合能力"]

**改写示范**：
- 原描述：[用户简历中的一段话]
- 战略化改写：[融入公司战略关键词后的专业描述]
- 改写逻辑：[解释为什么要这样改，如何体现了对公司痛点的解决能力]

**建议新增关键词**：[根据战略分析给出的核心词汇]

## STAR 深度提问
**背景**：[说明为什么要问这个问题]

**问题**：请向用户提5个 STAR 引导问题，请不要提问虚无的"学习能力"。请结合搜索到的公司痛点（如：生鲜损耗），设计一个【情境模拟题】："如果你面临 XX 业务问题，你会如何利用你简历中的 XX 技能去解决？"，并要求用户按 STAR 法则回答。
```

**输出格式**:
```typescript
interface InterviewAnalysisOutput {
  company_strategy: {
    business_status: string;    // 业务现状
    job_subtext: string;        // 岗位潜台词
  };
  ats_score: {
    score: number;              // 0-100
    advantages: string[];       // 优势点
    gaps: string[];             // 致命缺口
  };
  resume_suggestions: {
    logic: string;              // 修改逻辑
    rewrite_demo: {
      original: string;         // 原描述
      rewritten: string;        // 战略化改写
      reasoning: string;        // 改写逻辑
    };
    new_keywords: string[];     // 建议新增关键词
  };
  star_questions: {
    background: string;         // 提问背景
    questions: string[];        // 5道STAR问题
  };
}
```

## 7. 错误处理

### 7.1 错误码定义
| 错误码 | 说明 | 处理建议 |
|--------|------|----------|
| `RESUME_PARSE_ERROR` | 简历解析失败 | 提示用户检查文件格式 |
| `SEARCH_TIMEOUT` | 搜索超时 | 重试或降级到通用分析 |
| `LLM_RATE_LIMIT` | LLM 限流 | 指数退避重试 |
| `INSUFFICIENT_INFO` | 信息不足 | 提示用户补充更多简历内容 |

### 7.2 降级策略
- 当搜索插件不可用时，LLM-2 基于行业通用趋势进行分析
- 当 LLM-2 调用失败时，返回 LLM-1 的关键词结果 + 通用建议

## 8. 安全与隐私

1. **数据脱敏**：简历内容不持久化存储，仅用于本次分析
2. **传输加密**：全站 HTTPS
3. **API 密钥管理**：LLM API Key 通过环境变量注入，不暴露给前端
4. **搜索关键词过滤**：避免生成敏感或不当搜索词
